C++如何解析函数调用

Name lookup

Member name lookup, 当函数名位于.或是->右侧时使用Member name lookup. 例如foo->bar(), 该方式用于查找特定类成员.

Qualified name lookup, 当包含::时, 例如std::sort等显式指定namespace的, 将会在指定命名空间内寻找调用的函数.

Unqualified name lookup, 当编译器得到非限定名称时, 会根据上下文在不同的范围内寻找匹配的声明.

对于上例的blast(ast, 100);, 有3个候选函数, 第1个候选函数显示了一个特性argument-dependent lookup, ADL, 即使调用方并未指定namespace为galaxy, 编译器仍会在参数的命名空间中寻找候选的函数.

Special Handling of Function Templates

blast(ast, 1000)并未指定模板参数, 因此编译器需要使用模板参数推导的方式确定③中的类型T.

之后编译器将会进行模板参数替换.

在模板参数替换失败时, template function就会从候选列表中被删除. 在模板元编程中, 称为SFINAE, substitution faliure is not an error.

Overload Resolution 重载决议

在这一阶段函数模板已经消失, 只剩下一组整洁的待候选函数集合.

接下来的工作室进一步缩小这一集合, 确定那些函数可以处理该函数调用.

最明显的要求就是函数的参数列表需要兼容.

candidate①的第二个参数float可以由int隐式转换而来.

对于candidate②, 由于Target有一个转换构造函数Target(galaxy::Asteroid *ast) : ast{ast} {}, 因此第一个参数可隐式转换为Target类型, 但是参数个数不匹配. 可以使用explicit的方式进行显式标记避免隐式转换.

candidate③的参数列表与①一致, 因此也兼容.

Tiebreakers 决胜局

在上例中, 只剩下2个可供调用的函数, 但是他们的参数列表都符合条件, 能够处理原始的function call. 但是编译器必须确定哪个是最好的可行函数.

(1) Better-matching parameters wins / 参数匹配更高者获胜

C++最重视调用者的参数类型与函数参数类型的匹配程度, 总是选择从给定的参数进行较少隐式类型转换的函数. 例如对于std::vector的operator[], 将会决定调用const版本还是non const版本. 在示例中, 2个备选函数具有相同参数列表, 因此决策进入下一局.

(2) Non-template function wins / 非模板函数取胜

C++更倾向调用非模板函数而不是模板函数, 因此最终将会调用

(3) More specialized template wins / 更偏特化的模板函数取胜

第1个模板函数接收任意类型作为其第1个参数, 而第2个模板函数只接受指针类型, 因此第2个函数模板更特化. 因此, 对于上面2个函数, blast(ast, 1000)将会调用第2个而不是第1个.
